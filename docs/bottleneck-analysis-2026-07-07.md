# PrismRAG 项目瓶颈诊断报告

> 日期: 2026-07-07
> 方法: 基于三层评测（Layer 1 检索 / Layer 2 生成 / Layer 3 端到端 QA）+ 消融 + 历史 run 的实测数据，定位当前最制约系统效果的瓶颈。
> 性质: 纯诊断分析，不含修复代码。优先级与修复方案见后续路线图（另行产出）。

---

## 0. TL;DR — 一句话结论

**当前最大瓶颈不是"检索能不能找到"，而是"检索回来的上下文相不相关"。**

- Layer 1 检索排序已很强（Full+zerank2 **NDCG@10 = 0.5715**）。
- Layer 2 上下文相关性却极低（**Context Relevance = 0.076**，100-query 下也仅 0.116）。
- 这两个数字同时成立，说明：**目标页面大多进了 Top-K，但真正喂给 LLM 的句子大部分与问题无关。**
- 相关性缺失沿链路向下传导，导致端到端答案正确性只有 **0.64**。

> 一句话：系统"找对了文件、喂错了内容"。瓶颈在"上下文过滤/压缩"这一环，而不在检索召回。

## 2. 瓶颈定位（按严重度排序）

### 🔴 B0 — Context Relevance 极低（致命瓶颈）

- **现象**: CtxRel 0.076（283）/ 0.116（100）。即检索回的上下文中，**超过 90% 的句子与问题无关**。
- **证据链**:
  - L1 NDCG@10 = 0.5715（页面级排序好）→ 目标页进了 Top-K。
  - 但 L2 CtxRel = 0.076（句子级相关性差）→ 进来的页面里，真正有用的句子没被挑出来。
  - 二者矛盾，唯一解释：**chunk / 上下文压缩粒度太粗，把整页或整块无关文本一并喂入**。
- **根因（推断）**:
  1. 上下文压缩仅靠"BGE 句级 cosine > 0.4"过滤，阈值宽松 + 句子边界切分不准，大量无关句漏过。
  2. chunk 粒度（整页/大块）与 LLM 上下文窗口不匹配，相关句被无关句稀释。
  3. reranker 只对 Top-K 文档重排，**不保证句子级相关性**。
  4. TO 手册类文档噪音行（已做 6 步正则清洗，但未根治）持续污染。
- **影响**: 直接拉低 Faithfulness（无关上下文诱导 LLM 编造/跑题）与 Answer Correctness。这是所有下游指标的"天花板"。

### 🟠 B1 — 端到端答案正确性偏低（0.64）

- **现象**: 50 条可回答问题中 18 条错误（handoff 分解：6 合理拒答误判 + 5 检索缺失 + 4 数值错误 + 3 其它）。
- **与 B0 的关系**: 5 条"检索缺失"是 L1 长尾漏召；4 条"数值/计算"是 pipeline 不支持多步推理；其余与 B0 的上下文噪音同源。
- **影响**: 这是用户最终感知到的质量，是 B0 的下游症状。

### 🟡 B2 — Visual 路几乎失效 + 历史 OOM（投入产出严重倒挂）

- **现象**: Visual_only NDCG@10 = 0.1564，对整体 NDCG 增量贡献很小；却占用 2 GB 索引 + 4 GB 缓存，且曾在 4090（24 GB）上 CUDA OOM（ColPali 常驻 11.4 GB + MaxSim 21 GB）。
- **证据**: 换 ColQwen2 仍仅 0.1564，说明非模型问题；`handoff` 记录"Visual 路距官方 ColPali 差 3.6x"，根因在管道（query 编码 / attention_mask / 评分）。
- **影响**: 高成本、低收益、且有稳定性风险。当前阶段它是"拖累项"而非"增益项"。

### 🟡 B3 — 评测口径不稳定，小样本高估（决策风险）

- **现象**: Faithfulness 在 50→100→283 条从 0.887 → 0.889 → **0.772**，CtxRel 在 100→283 从 0.116 → 0.076。
- **影响**: 若以 50/100 条结果做技术决策，会系统性高估 ~0.11。这不直接降低系统质量，但会**误导瓶颈判断与优先级投入**。

### 🟢 B4 — 工程效率 / 部署风险（次要）

- 端到端延迟 2.19 s（含 Ollama qwen2:7b HyDE + rerank），本地 macOS 无法跑全量（FAISS HNSW segfault、无本地 PG、Visual OOM），强依赖云端 GPU 按小时计费。
- 影响: 迭代成本高、复现慢，但不直接决定效果上限。

## 3. 核心矛盾图解

```
                 检索层 (L1)              生成层 (L2)              端到端 (L3)
  指标:      NDCG@10 = 0.57 ✅      CtxRel = 0.076 ❌        Correctness = 0.64 ❌
  含义:      页面找得对              句子挑不准               答案答不对

  ┌──────────┐    ┌──────────────────┐    ┌──────────────┐    ┌──────────┐
  │  Query    │───▶│ 三路检索+RRF+rerank │──▶│ 上下文压缩/过滤 │──▶│  LLM 生成 │
  │          │    │  (强)            │    │  (弱 ←瓶颈)  │    │  (被污染) │
  └──────────┘    └──────────────────┘    └──────────────┘    └──────────┘
                        ↑ 这一环工作良好           ↑ 这一环是瓶颈
```

**关键推论**: 在 L1 已经 0.57 的前提下，继续堆检索模型（换视觉 backbone、加 HyDE）的边际收益极小（HyDE 实验已证无效）。**下一个数量级的提升只能来自"上下文相关性"这一环**——即更准的句子级过滤 / 更优的 chunk 策略 / 更好的压缩。

## 4. 什么"不是"瓶颈（防止误投入）

| 疑似瓶颈 | 实际判断 | 依据 |
|---------|---------|------|
| 检索召回能力不足 | ❌ 不是 | NDCG@10=0.57，目标页大多进 Top-K |
| 拒答策略差 | ❌ 不是 | Rejection Accuracy=0.95，20 条域外全正确拒答 |
| 幻觉严重 | ❌ 不是 | 严重幻觉率仅 2%，Faithfulness 0.77 主因是上下文噪音而非编造 |
| 视觉多模态价值 | ⚠️ 暂未兑现 | Visual_only 0.156，成本高收益低，先修管道再谈增益 |
| 评测体系缺失 | ❌ 不是 | 三层评测 + 12 路消融已齐备，问题在口径稳定性(B3) |

## 5. 优先级结论

| 优先级 | 瓶颈 | 为什么先处理 |
|:--:|------|------------|
| **P0** | **B0 Context Relevance** | 它是所有下游指标的"天花板"，且 L1 已封顶，唯一能撬动数量级提升的环节 |
| P1 | B1 答案正确性长尾（检索缺失 + 数值类） | B0 修复后的剩余长尾，需针对性补召回/拒答规则 |
| P1 | B3 评测口径稳定（统一到 283 全量） | 防止决策被小样本误导 |
| P2 | B2 Visual 路管道修复 / 或降级 | 高成本低收益，先验证管道根因再决定是否保留 |
| P2 | B4 工程效率 | 影响迭代速度，不影响效果上限 |

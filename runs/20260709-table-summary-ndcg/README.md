## 干净隔离：把"视觉编码器"这个变量消掉

Dense/BM25 是文本路（BGE/BM25），**视觉编码器无关**；Visual 路只看页面图，分块无关。三条可干净归因：

| 路 | 旧分块(20260702/04) | 表格摘要分块(当前) | 解读 |
|----|---------------------|-------------------|------|
| **Dense_only** (BGE文本) | 0.3938 | 0.3638 | **-0.0300 (-7.6%)** = 纯"摘要替代整表编码"效应 |
| **BM25_only** (词法) | 0.4432 | 0.4248 | **-0.0184 (-4.1%)** = 纯分块效应（摘要关键词比整表少） |
| **Visual_only** (ColQwen2) | 0.1564 (20260704) | 0.1590 | **+0.0026 ≈ 持平** = 分块不影响视觉检索 ✅ |

→ 结论：**表格摘要分块确实削弱了纯文本检索信号**（Dense/BM25 降），但**视觉检索不受影响**（符合预期，页面图与 chunk 文本正交）。

## 为什么 NDCG 整体还跌了 6%？

20260704 已验证 **ColQwen2 视觉(0.1564) > ColPali-v1.3(~0.1365)**，即"视觉编码器升级"这步是**加分项**。所以 NDCG 整体从 0.5507→0.5162 的 -0.0345，**主要来自分块/摘要对文本路的削弱**（Dense -0.030、BM25 -0.018、BM25_Dense_Visual -0.019，与 ColQwen2 基线 20260704 比），视觉升级的加分被文本路掉分盖过。

## 结论（NDCG + RAGAS 完整 verdict）

1. **NDCG 不是本特性的目标指标**。设计 §10.4 明确表格摘要要动的是 Faithfulness/AnswerRelevancy，不是检索排序。文本路 NDCG 小幅下降（Dense -7.6%、BM25 -4.1%）是**预期权衡**：摘要替代整表编码丢了字面单元格信号，换来"超大表不被切碎、可检索+展开"。
2. **RAGAS 印证了生成端收益（特性主目标达成）**：Faithfulness 0.901（>0.90，比旧分块 +1.7pp）、AnswerRelevancy 0.804（+0.7pp）、ContextRelevancy 0.263（翻倍 +14.5pp）。答案更忠实、上下文更切题。
3. 视觉路 NDCG 完全不受影响（持平），ColQwen2 升级是净加分。
4. **整体判定**：表格摘要 + 大表保护"实现后首次云端全量"——检索侧付出预期内的小幅代价，生成侧拿到设计预期的正收益。特性方向正确，值得保留。若要进一步压低文本路 NDCG 损失，可考虑"Dense 同时编码摘要+关键单元格"的混合 embed_text。
